Service Unavailable

The platform default store is not configured yet.

What does this mean?

The Wasla e-commerce platform requires a default store to be configured for the root domain. This store acts as the landing page for visitors accessing the platform.

How to fix this?

An administrator needs to create a platform default store (or mark an existing store as the platform default).

You can create this store by:

  1. Running: python manage.py ensure_platform_store
  2. Using the admin panel at /admin/ to set is_platform_default
  3. Optionally setting WASSLA_PLATFORM_STORE_SLUG (current: {{ default_store_slug }})

For technical support or to configure the default store, please contact your administrator.
