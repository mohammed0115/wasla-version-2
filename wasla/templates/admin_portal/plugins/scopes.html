About Permission Scopes

Permission scopes define what resources and actions the plugin is allowed to access. Use a deny-by-default security model:
- Each scope must be explicitly granted
- Use hierarchical naming (e.g., plugin.data.read)
- Document what each scope allows
Scopes are enforced when the plugin is enabled and when it receives events.
